# 👥 РОЛИ ПОЛЬЗОВАТЕЛЕЙ В ПРИЛОЖЕНИИ IAPM

## 🎯 ОБЩАЯ ИНФОРМАЦИЯ

Приложение IAPM поддерживает **три роли пользователей** с разными интерфейсами и возможностями:

- **👤 Пацієнт** - человек, который принимает лекарства
- **👨‍⚕️ Опекун (Сиделка)** - ухаживает за пациентами
- **🏥 Лікар** - лечит пациентов и мониторит их состояние

---

## 👤 РОЛЬ: ПАЦІЄНТ

### Что видит пациент:
```
🏠 Главный экран:
├── 📅 Сегодняшние лекарства (с кнопками "Принять")
├── 📊 Статистика (принято/пропущено/прихильність)
├── ⏰ Ближайшие напоминания
└── ➕ Кнопка "Добавить лекарство"

⚙️ Настройки:
├── Личные данные (имя, дата рождения, фото)
├── Темная тема и другие настройки интерфейса
├── Уведомления о приеме лекарств
└── История приема
```

### Основные действия:
- ✅ Отмечать принятые лекарства
- ✅ Просматривать статистику приверженности лечению
- ✅ Получать напоминания о приеме
- ✅ Добавлять новые лекарства
- ✅ Работать оффлайн (без интернета)

---

## 👨‍⚕️ РОЛЬ: ОПЕКУН (СИДЕЛКА)

### Что видит опекун:
```
🏠 Главный экран:
├── 👥 Список пациентов под опекой
├── 📊 Общая статистика по всем пациентам
├── ➕ Кнопка "Добавить пациента"
└── 📈 Аналитика по пациентам

👤 Экран пациента:
├── 💊 Лекарства пациента
├── 📊 Статистика adherence
├── ⏰ Последний прием лекарств
└── 📞 Контактная информация
```

### Основные действия:
- 👀 **Мониторить** прием лекарств пациентами
- 📞 **Связываться** с пациентами
- ➕ **Добавлять** новых пациентов под опеку
- 📊 **Просматривать** отчеты о приверженности
- ⚠️ **Получать уведомления** о пропущенных дозах

---

## 🏥 РОЛЬ: ЛІКАР

### Что видит врач:
```
🏠 Главный экран:
├── 👥 Список пациентов с диагнозами
├── ⚠️ Уровень риска каждого пациента
├── 📊 Статистика по всем пациентам
└── 📋 Аналитические отчеты

👤 Карточка пациента:
├── 📋 Диагнозы и состояние здоровья
├── 💊 Назначенные лекарства
├── 📊 График приверженности лечению
├── ⚠️ Уровень риска (Высокий/Средний/Низкий)
└── 📅 Дата последнего визита
```

### Основные действия:
- 📊 **Анализировать** приверженность пациентов лечению
- ⚠️ **Определять** пациентов с высоким риском
- 📋 **Просматривать** медицинские данные
- 📈 **Генерировать** отчеты для анализа
- 🔔 **Получать уведомления** о проблемах с adherence

---

## 🔄 ВЗАИМОСВЯЗЬ РОЛЕЙ

```
Пацієнт ← Опекун ← Лікар
    ↑         ↑         ↑
  Приймає   Стежить   Лікує
  ліки     за прийомом  і аналізує
```

### Пример рабочего процесса:
1. **Лікар** назначает лекарства пацієнту
2. **Пацієнт** получает напоминания и отмечает прием
3. **Опекун** следит за соблюдением графика
4. **Лікар** анализирует данные и корректирует лечение

---

## 📱 ТЕХНИЧЕСКИЕ ОСОБЕННОСТИ

### Для всех ролей:
- ✅ **Оффлайн-режим** - работает без интернета
- ✅ **Синхронизация** - данные сохраняются в облаке
- ✅ **Push-уведомления** - напоминания о приеме
- ✅ **Material 3** - современный дизайн

### Адаптация интерфейса:
- 📱 **Пацієнт**: Упрощенный интерфейс для пожилых людей
- 👨‍⚕️ **Опекун**: Инструменты мониторинга и связи
- 🏥 **Лікар**: Аналитические дашборды и отчеты

---

## 🚀 ЗАПУСК ДЛЯ РАЗНЫХ РОЛЕЙ

### Тестирование пациента:
```kotlin
// При регистрации выберите роль "Пацієнт"
role = UserRole.PATIENT
```

### Тестирование опекуна:
```kotlin
// При регистрации выберите роль "Опекун"
role = UserRole.CAREGIVER
```

### Тестирование врача:
```kotlin
// При регистрации выберите роль "Лікар"
role = UserRole.DOCTOR
```

---

## 📊 СТАТИСТИКА И ОТЧЕТЫ

### Для пациентов:
- 📈 График приверженности за неделю/месяц
- ✅ Количество принятых/пропущенных доз
- 🎯 Процент соблюдения графика

### Для опекунов:
- 👥 Общая статистика по всем пациентам
- ⚠️ Пациенты с низкой приверженностью
- 📞 Контакты для быстрой связи

### Для врачей:
- 📋 Отчеты по adherence для каждого пациента
- ⚠️ Список пациентов с высоким риском
- 📊 Тренды и аналитика по группам пациентов

## 🎨 ДИЗАЙН И UX

### Адаптивность:
- 📱 **Пацієнти**: Крупные кнопки, простая навигация
- 👨‍⚕️ **Опекуни**: Эффективные списки и быстрый доступ

### Цветовая кодировка:
- 🟢 **Зеленый** - хорошая приверженность
- 🟡 **Желтый** - средняя приверженность
- 🔴 **Красный** - низкая приверженность/высокий риск

## 🔒 БЕЗОПАСНОСТЬ И КОНФИДЕНЦИАЛЬНОСТЬ

### Row Level Security (RLS):
- 👤 Пацієнт видит только свои лекарства
- 👨‍⚕️ Опекун видит только своих пацієнтів
- 🏥 Лікар видит только своих пацієнтів

### Доступ к данным:
- 📱 Локальное хранение (SQLite)
- ☁️ Синхронизация с Supabase
- 🔐 Шифрование чувствительных данных
